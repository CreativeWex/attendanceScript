# AttendanceScript: автоматизация учета посещаемости и отчетности по сотрудникам

Пример запуска
`python3 -m generate_report --input-dir examples --output output/report.xlsx`

## Как пользоваться

## Подготовьте входную папку

Скрипт получает данные из каталога `--input-dir` и обрабатывает файлы в нём, результат записывается в файл `--output`

В папке должны быть:

1. файлы с данными посещаемости со СКУД;
2. файл режима работы (для заполнения данных о режиме работы сотрудника - "офисный труд", "ПУРР" и т.д.).
3. файл с расписанием работы (для заполнения данных о времени начала и конца рабочего дня).

### Общие файлы посещаемости (приход/уход) со СКУД

Поддерживаются 2 типа входных данных:

#### `.xls` (legacy)

- имя файла содержит ФИО сотрудника (например, `Иванов.xls` или `Фамилия Имя Отчество.xls`);
- данные читаются из первого столбца:
  - даты: `dd.mm.yyyy` (или Excel-датой);
  - времена: `HH:MM` / `HH:MM:SS` (или Excel-временем).

#### `.xlsx` (табличный формат)

По заголовкам скрипт ищет:

- колонку фамилии (ключи: `фамилия/lastname`);
- колонку имени (ключи: `имя/firstname`);
- (опционально) колонку отчества (ключ: `отчество/middlename`);
- колонку(и) времени событий (ключи: `EventTime`, `ReceiptTime`, а также варианты с "ВремяПКБ/ВремяПОКБ" и подобными).
Если есть колонка направления `EntryExit` со значениями `Вход` / `Выход`, она используется для расчёта отсутствий в течение дня.

### Файл режима работы (для заполнения данных о режиме работы сотрудника - "офисный труд", "ПУРР" и т.д.)

Поддерживаются следующие справочники:

- `Режимы работы.xlsx` или `Режим работы.xlsx`

Обязательные поля:

- `ФИО`
- `Режим работы`

### Файл с расписанием работы (для заполнения данных о времени начала и конца рабочего дня)

Поддерживаются следующие справочники:

- `Расписание работы.xlsx`

#### `Режимы работы.xlsx` / `Режим работы.xlsx`

Нужные поля (названия могут отличаться по пунктуации; скрипт нормализует заголовки):

- `Ф.И.О. сотрудника`
- `НачалоНормРабочВрем`
- `КонецНормРабчВр` также распознаётся вариант `КонецНормРабочВр`

Дополнительное правило для `Расписание работы.xlsx`:

- если `НачалоНормРабочВрем` распарсился как `0:00:00`, в отчёте начало считается как `09:00`, и ячейка "Начало рабочего дня" окрашивается в жёлтый;
- если `КонецНормРабчВр` распарсился как `0:00:00`, в отчёте конец считается как `18:00`, и ячейка "Конец рабочего дня" окрашивается в жёлтый.

## Запуск

Из корня проекта:
`python3 -m attendance_report.cli --input-dir examples --output output/report.xlsx`

## Легенда цветов и что именно раскрашивается в Excel

Ниже цвета из кода (`report_writer.py`) и какие ячейки они подсвечивают.

### 1) `WEEKEND_GREY_FILL` (серый) - выходные дни

- Раскрашиваются **все ячейки в колонках дат** в строках дневной сетки, если дата помечена как `Выходной` (суббота/воскресенье).

### 2) `YELLOW_FILL` (желтый) - подстановка/фолбэк

Желтый появляется в двух случаях:

- В `Начало рабочего дня` (колонка `B`): если в `Расписание работы.xlsx` значение `НачалоНормРабочВрем` распарсилось как `00:00:00` (считаем как `09:00`).
- В `Конец рабочего дня` (колонка `C`): если в `Расписание работы.xlsx` значение `КонецНормРабчВр` распарсилось как `00:00:00` (считаем как `18:00`).
- Также желтым раскрашивается ячейка `Время ухода` в дневной сетке, если `Конец рабочего дня` был подставлен как фолбэк `18:00` из-за отсутствия парного выхода (`departure_time_fallback=True`).

### 3) Зеленые/красные подсветки по переработкам и отклонениям

Эти правила применяются только на будни и только там, где есть значения.

`LIGHT_GREEN_FILL`:

- В строке `Отклонение по времени прихода`: `факт приход < план приход` более чем на 15 минут.
- В строках `Переработка` и `Переработка факт (с учетом отсутствия ...)`: `0 < diff <= 30 минут` (и значение не считается "нулевым").

`DARK_GREEN_FILL`:

- В строках `Переработка` и `Переработка факт (с учетом отсутствия ...)`: `diff > 30 минут`.

`LIGHT_RED_FILL`:

- В строке `Отклонение по времени прихода`: `факт приход > план приход` более чем на 15 минут.
- В строках `Переработка` и `Переработка факт (с учетом отсутствия ...)`: `-30 минут <= diff < 0` (и значение не близко к нулю).

`DARK_RED_FILL`:

- В строках `Переработка` и `Переработка факт (с учетом отсутствия ...)`: `diff < -30 минут`.

### 4) Подсветка по "низким значениям" длительности

Дополнительные правила:

- Для строки `Длительность факт` по дням (только будни):
  - `08:00 < время < 09:00` → `LIGHT_RED_FILL`
  - `время < 08:00` → `DARK_RED_FILL`
- Для колонки `Среднее время работы`:
  - `08:00 < среднее < 09:00` → `LIGHT_RED_FILL`
  - `среднее < 08:00` → `DARK_RED_FILL`

## Поведение при отсутствии информации и значения по умолчанию

### 1) Нет файла c режимом работы

- Если в папке нет `Режимы работы.xlsx`/`Режим работы.xlsx` и `Расписание работы.xlsx`:
  - в `Режим работы` в отчёте ставится `Не найден файл режима работы`;
  - плановые времена берутся по умолчанию:
    - `Начало рабочего дня` =`09:0`;
    - `Конец рабочего дня` = `18:00`.

### 2) Нет строки для конкретного сотрудника в справочнике

- В `Режим работы` ставится `Информация по данному сотруднику не найдена`.
- При этом плановые времена всё равно подставляются по умолчанию (начало = `--official-time`, конец = `18:00`).

### 3) Как формируется `18:00` при отсутствии "парного" выхода

- В `aggregator.py` фолбэк `departure_time=18:00` включается, если:
  - на буднях после последнего входа нет выхода (в логике с направлением `EntryExit`),
  - или в legacy-логике последовательность заканчивается "входом" без парного "выхода".
- В отчёте это дополнительно подсвечивается желтым в дневной ячейке `Время ухода`.

### 4) Что выводится при пропуске файлов

- Скрипт обходит все файлы в `--input-dir`.
- Файлы режима/расписания и временные Excel-файлы `~$*.xlsx` игнорируются.
- Если файл посещаемости не соответствует ожидаемой структуре - он пропускается, а причина выводится в `Skipped file details` в консоли.
